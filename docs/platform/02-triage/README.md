# IsoStack Platform Triage

This folder records ownership, severity, architecture and disposition decisions for
Platform CR inputs and findings.

Triage should result in one explicit outcome:

- rejected or already resolved;
- consolidated into an existing control;
- deferred to the platform assurance/refinement register;
- returned for a missing business/technical decision;
- reassigned to Commerce or a module lane; or
- promoted as a candidate for bounded Platform slice planning.

Triage does not confirm implementation and does not bypass root-roadmap sequencing.
