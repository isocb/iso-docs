# IsoStack Platform Change-Request Inputs

Place Platform change requests, briefs, reported defects and cross-cutting findings
here before triage.

A CR input captures need and evidence; it does not authorise implementation. Clearly
separate settled decisions, open questions, assumptions and non-goals.

Suggested name:

```text
YYYY-MM-DD-isostack-core-platform-<topic>-cr.md
```

If a concern was discovered through a module, record that provenance while assigning
ownership according to the affected capability rather than the route/file location alone.
